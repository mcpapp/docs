These examples focus on how the public components fit together.

For where specs come from, use [Spec](/specs/). That guide covers hand-authored JSON, database-backed specs, generated specs, streams, and source selection.

## Examples

- [Native MCP App](/examples/native-mcp-app/) shows the smallest app view for native MCP Apps hosts.
- [Standalone Web](/examples/web-host/) shows a browser page using `WebHost`.
- [WebHost SSR](/examples/webhost-ssr/) shows a server-rendered browser first paint.
- [Next App SSR](/examples/next-ssr/) shows upstream `@json-render/next` for full Next.js apps.
- [Custom Registry](/examples/custom-registry/) shows how to add product components.
- [Typed Stack](/examples/typed-stack/) shows a typed customer profile flow from catalog to validated spec.
- [Actions + XState](/examples/xstate-actions/) shows action handlers with a state machine.
- [Sync Store](/examples/sync-store/) shows how synced server records can flow through the same `state` prop.
- [Local Server](/examples/local-server/) shows stdio server wiring with upstream json-render and MCP packages.
- [HTTP Server](/examples/http-server/) shows Streamable HTTP server wiring.
- [Static Serving](/examples/static-serving/) shows how to build and serve the MCP App HTML resource.
